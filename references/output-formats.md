# Engagement Types & Output Formats

## Engagement Type Guide

| Type | Lead Agents | Key Deliverables |
|---|---|---|
| **Market Entry Analysis** | Strategist, Associate | Market sizing, Five Forces, PESTLE, financial projections, recommendation |
| **Competitive Intelligence** | Strategist, Associate, Tech Specialist | Competitor profiles, VRIO, positioning map |
| **Due Diligence (M&A)** | Financial Analyst, Tech Specialist, Strategist | Financial model, VRIO, tech assessment, risk register |
| **Strategy Review** | Strategist (AFI), Partner | Full AFI, recommendations, implementation roadmap |
| **Business Case** | Financial Analyst, Strategist | P&L model, NPV/IRR, risk analysis, recommendation |
| **Go-to-Market** | Strategist, Tech Specialist | Crossing the Chasm, segment analysis, GTM roadmap |
| **Operational Review** | Financial Analyst, Strategist | Value chain, benchmarking, cost reduction opportunities |
| **Technology Assessment** | Tech Specialist, Financial Analyst | Tech landscape, build/buy/borrow, cost model |

## Output Format Standards

### Executive Summary (1 page)
```
EXECUTIVE SUMMARY
[Client / Engagement Name] | [Date]

SITUATION: [2-3 sentences — what is the decision or problem?]

FINDING: [The single most important insight — the "so what"]

IMPLICATIONS:
• [Implication 1 — with magnitude if possible]
• [Implication 2]
• [Implication 3]

RECOMMENDATION: [Clear, actionable, specific]

FINANCIAL IMPACT: [Range, timeframe, key assumption]

RISK: [Top 1-2 risks to the recommendation]

NEXT STEPS: [3-5 actions with owners and dates]
```

### Strategy Memo
```
TO: [Audience]
FROM: [Team]
DATE: [Date]
RE: [Subject]

BOTTOM LINE: [1 sentence answer to the question]

CONTEXT: [2-3 sentences max — why this matters now]

ANALYSIS: [Organized by framework; each section ends with "so what"]

OPTIONS CONSIDERED:
Option A: [Describe] | Pros: [...] | Cons: [...] | Financial impact: [...]
Option B: [Describe] | Pros: [...] | Cons: [...] | Financial impact: [...]

RECOMMENDATION: [Option X because Y and Z]

IMPLEMENTATION: [Key steps, timeline, owners]

RISKS & MITIGATIONS: [Table format preferred]

SOURCE INDEX: [See Source Index section]
```

### Slide Structure (for presentation deliverables)
Each slide follows:
- **Title**: The insight (not the topic). Bad: "Market Analysis." Good: "Market is growing 18% YoY but profitability is declining — suggesting commoditization risk."
- **Body**: Evidence supporting the title
- **So What box** (bottom right or below): "This means [implication for client]"
- **Source footnote**: [SRC-XXX] for every data point on the slide

### Financial Model Output
- Tab 1: Summary Dashboard (KPIs, charts, recommendation)
- Tab 2: Assumptions (all inputs, all sourced)
- Tab 3: P&L Projection (3/5/10 year)
- Tab 4: Balance Sheet (if applicable)
- Tab 5: Cash Flow Statement
- Tab 6: Valuation (if applicable)
- Tab 7: Sensitivity Analysis
- Tab 8: Scenarios (Bear/Base/Bull)
- Tab 9: Source Index

## Formatting Rules (all deliverables)

1. **Numbers**: Use consistent formatting ($X.Xm / $X.Xbn; %; X.Xx multiple)
2. **Dates**: Always specific (Q3 2023, not "recently"; FY2023, not "last year")
3. **Geography**: Always stated (US, North America, Global — never assumed)
4. **Currency**: Always labeled ($USD unless stated)
5. **Real vs. nominal**: Always labeled ("constant 2024 USD" or "nominal")
6. **Acronyms**: Define on first use
7. **Source tags**: `[SRC-XXX]` immediately after the claim, before the period
8. **Flagged items**: `[CITATION NEEDED]`, `[HALLUCINATION RISK]`, `[UNVERIFIED]` in bold red (or capitalized) — these must be resolved before delivery
